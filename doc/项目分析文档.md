## 一、项目概述

### 1.1 项目愿景

财务数据管理系统是一个基于 Spring Boot 3.2 + Java 21 的多模块 Maven 项目，旨在提供完善的财务数据管理能力。当前项目已完成基础架构搭建，实现了用户认证、权限管理等核心功能，财务业务模块待进一步开发。

### 1.2 技术栈总览

|------|------|------|
| 核心框架 | Spring Boot | 3.2.2 |
| Java 版本 | Java | 21 |
| 构建工具 | Maven | - |
| 安全框架 | Spring Security + Sa-Token | 1.37.0 |
| ORM 框架 | MyBatis Plus | 3.5.5 |
| 数据库 | MySQL | 8.0 |
| 连接池 | Druid | 1.2.20 |

## 五、核心业务逻辑

### 5.1 认证授权流程

**架构设计**：
- 采用 Spring Security + Sa-Token 双重认证机制
- Spring Security 负责基础的 HTTP 安全过滤
- Sa-Token 负责细粒度的权限控制

**核心组件**：
| 组件 | 说明 |
|------|------|
| `SecurityConfig` | Spring Security 配置，白名单设置 |
| `UserDetailsServiceImpl` | 实现 UserDetailsService，加载用户权限 |
| `AuthService` | 认证服务，处理登录、注册、验证码 |
| `PermissionAspect` | 权限切面，处理权限注解 |

**登录流程**：
```
1. 用户请求登录（携带手机号、密码、验证码）
2. 验证码校验（从 Redis 获取）
3. 用户查询（如果不存在则自动注册）
4. 密码验证（MD5 + salt）
5. Sa-Token 登录，生成会话
6. 更新最后登录时间
7. 返回 token 和用户信息
```

**注册流程**：
```
1. 分布式锁防止并发注册
2. 验证码校验
3. 检查手机号是否已注册
4. 生成唯一用户ID（布隆过滤器去重）
5. 设置密码盐值
6. 保存用户信息
7. 返回注册结果
```

### 5.2 用户管理

**用户状态枚举**：
| 状态 | 说明 |
|------|------|
| INIT | 初始状态 |
| AUTH | 已认证 |
| FROZEN | 冻结 |

**用户角色枚举**：
| 角色 | 说明 |
|------|------|
| CUSTOMER | 普通用户 |
| ADMIN | 管理员 |
| SUPERADMIN | 超级管理员 |

**核心业务方法**：
- `register()` - 用户注册
- `auth()` - 实名认证
- `active()` - 激活用户
- `login()` - 登录验证

### 5.3 权限控制

**权限注解**：
| 注解 | 说明 |
|------|------|
| `@RequiresPermissions` | 基于权限点控制 |
| `@RequiresRoles` | 基于角色控制 |

**权限检查逻辑**：
- 支持 AND/OR 逻辑组合
- 方法级注解权限控制
- 自定义 `PermissionAspect` 处理

### 5.4 缓存策略

**JetCache 配置**：
- 本地缓存：Caffeine
- 远程缓存：Redis
- 缓存 key 格式：`:user:cache:id:{userId}`
- 缓存时间：2 小时

**缓存使用场景**：
- 用户信息缓存
- 验证码存储
- 权限数据缓存

### 5.5 分布式锁

**基于 Redisson 实现**：
- 注解：`@DistributeLock`
- 支持固定 key 和 SpEL 表达式
- 自动续期和释放

**使用示例**：
```java
@DistributeLock(scene = "USER_REGISTER", keyExpression = "#telephone")
public BaseResult<?> register(UserRegisterRequest request) {
    // 业务逻辑
}
```

### 5.6 数据加密

**字段级加密**：
- `AesEncryptTypeHandler` - MyBatis 类型处理器
- 加密字段：`realName`（真实姓名）、`idCardNo`（身份证号）

**密码加密**：
- 算法：MD5 + salt
- 每个用户独立的 salt 值

**敏感数据脱敏**：
- 手机号使用 `@SensitiveStrategyPhone` 注解脱敏
