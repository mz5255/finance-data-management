## 运行测试

### 1. 运行所有测试

```bash
mvn test
```

### 2. 运行指定模块的测试

```bash
# 运行 common 模块测试
mvn test -pl finance-common

# 运行 datasource 模块测试
mvn test -pl finance-datasource

# 运行 auth 模块测试
mvn test -pl finance-module/module-auth

# 运行 application 模块测试
mvn test -pl finance-application

# 运行 cache-starter 模块测试
mvn test -pl finance-starter/cache-starter
```

### 3. 运行指定测试类

```bash
mvn test -Dtest=AesUtilTest
mvn test -Dtest=UserServiceTest
mvn test -Dtest=AuthServiceTest
mvn test -Dtest=AuthControllerTest
mvn test -Dtest=RedisUtilsTest
```

### 4. 运行指定测试方法

```bash
# 运行单个测试方法
mvn test -Dtest=AesUtilTest#testEncryptDecrypt

# 运行多个测试方法（使用 * 通配符）
mvn test -Dtest=AesUtilTest#testEncrypt*
```

### 5. 跳过测试

```bash
mvn clean package -DskipTests
```

## 测试规范

### 1. 测试类命名规范

- 测试类名：`{被测试类名}Test`
- 例如：`AesUtilTest`、`UserServiceTest`

### 2. 测试方法命名规范

- 测试方法名：`test{方法名}_{场景}`
- 例如：`testGetByTelephoneExists`、`testRegisterSuccess`

### 3. 测试注解使用

| 注解 | 用途 |
|------|------|
| `@Test` | 标记测试方法 |
| `@DisplayName` | 测试显示名称 |
| `@BeforeEach` | 每个测试方法执行前执行 |
| `@BeforeAll` | 所有测试方法执行前执行一次 |
| `@AfterEach` | 每个测试方法执行后执行 |
| `@ExtendWith(MockitoExtension.class)` | 启用 Mockito |
| `@Mock` | 创建 Mock 对象 |
| `@InjectMocks` | 自动注入 Mock 对象 |

### 4. 断言使用

```java
// 相等性断言
assertEquals(expected, actual);
assertNotEquals(expected, actual);

// 布尔断言
assertTrue(condition);
assertFalse(condition);

// 空值断言
assertNull(object);
assertNotNull(object);

// 异常断言
assertThrows(ExceptionClass.class, () -> {
    // 执行会抛出异常的代码
});
```

## 示例测试代码

### 工具类测试示例

```java
@Test
@DisplayName("测试加密解密正常流程")
void testEncryptDecrypt() {
    // 准备测试数据
    String originalText = "这是一个测试文本";

    // 执行测试
    String encryptedText = AesUtil.encrypt(originalText);
    String decryptedText = AesUtil.decrypt(encryptedText);

    // 验证结果
    assertNotNull(encryptedText);
    assertNotEquals(originalText, encryptedText);
    assertEquals(originalText, decryptedText);
}
```

### Service 层测试示例（使用 Mockito）

```java
@ExtendWith(MockitoExtension.class)
class UserServiceTest {

    @Mock
    private UserMapper userMapper;

    @InjectMocks
    private UserServiceImpl userService;

    @Test
    @DisplayName("测试根据手机号查询用户")
    void testGetByTelephone() {
        // 准备测试数据
        UserDO testUser = new UserDO();
        testUser.setTelephone("13800138000");

        // 设置 Mock 行为
        when(userMapper.selectOne(any(LambdaQueryWrapper.class)))
            .thenReturn(testUser);

        // 执行测试
        UserDO result = userService.getByTelephone("13800138000");

        // 验证结果
        assertNotNull(result);
        assertEquals("13800138000", result.getTelephone());
        verify(userMapper, times(1)).selectOne(any());
    }
}
```

## 注意事项

### 1. 依赖关系

确保父 `pom.xml` 中包含以下依赖：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

该依赖包含了：
- JUnit 5
- Mockito
- AssertJ
- Hamcrest
- JsonPath

### 2. 测试隔离

每个测试方法应该相互独立，不依赖执行顺序。使用 `@BeforeEach` 初始化测试数据。

### 3. Mock 使用

对于外部依赖（如数据库、Redis、HTTP 调用），使用 Mockito 进行 Mock，避免实际调用。

### 4. 异常测试

使用 `assertThrows` 测试异常场景：

```java
assertThrows(IllegalArgumentException.class, () -> {
    IDUtils.generateUniqueId(null);
});
```

## 常见问题

### 1. 测试运行失败

- 检查依赖是否正确引入
- 检查测试类包名是否与被测试类一致
- 查看错误日志确定具体问题

### 2. Mock 对象不生效

- 确保使用 `@ExtendWith(MockitoExtension.class)`
- 确保使用 `@Mock` 和 `@InjectMocks` 注解
- 检查 Mock 设置的参数匹配

### 3. Spring 上下文加载失败

- 确保使用 `@SpringBootTest` 的测试类可以正确加载应用上下文
- 检查配置文件是否完整
